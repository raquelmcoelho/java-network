Création d’un projet Web Java EE

Table des matières

1.      Télécharger un serveur d’application. 1

2.      Création du projet 1

3.      Vérifier la structure du projet 1

4.      Configuration de lancement 1

4.1.       Ajouter une configuration. 1

4.2.       Modifier l’URL dans la configuration de lancement 1

 

1.    Télécharger un serveur d’application

Choisir un serveur d’application qui va exécuter l’application Web.

On propose Apache Tomcat 9 ou 10, téléchargement ici : Apache Tomcat® - Which Version Do I Want? : pas d’installation, juste le fichier « core : zip » à télécharger et décompresser.

Autre possibilité : Apache TomEE 9 qui utilise les nouvelles bibliothèque Jakarta mais est beaucoup plus rigoureux (et donc plus pénible à paramétrer) : Apache TomEE

2.    Création du projet

Dans IntelliJ, aujourd’hui c’est le menu : New Project > Jakarta EE

Dans les anciennes versions : New Project > Java EE


Il est possible que la construction du projet ne fonctionne pas bien si le nom du projet (et donc le répertoire de projet) contient des espaces, de même que le chemin d’accès au projet ! (Méfiance avec certains caractères spéciaux)

Paramétrer « application server » via le bouton « New » et en sélectionnant le répertoire où vous avez dézipper Tomcat


Choix des dépendances comme précisé dans le sujet de TP : Persistence (JPA), Servlet et Hibernate


3.    Vérifier la structure du projet

Vérifier que la structure est bonne avec le menu « File > Project Structure »

On doit trouver dans « Module » et « Facets » une partie nommée Web

Ainsi que l’artéfact « war exploded » qui est ce que IntelliJ exécutera  (le « war » est un fichier déployable sur un serveur tierce).


Côté code, on doit trouver une classe « HelloServlet » dans « src/main/java » de même qu’un fichier « index.jsp »

Si la structure n’est pas bonne, il vaut mieux en général recréer le projet.

4.    Configuration de lancement

Si vous aviez un serveur au démarrage, le lancement est prêt.

4.1. Ajouter une configuration

Sinon vous pouvez ajouter une configuration de type « Tomcat (local) » :


Dans « application server », on peut de nouveau ajouter un serveur (cf. création du projet).

4.2. Modifier l’URL dans la configuration de lancement

Le but est ici de déployer l’application avec une URL particulière.


Remarque pour les serveurs TomEE : Tomcat ne pose pas de problème mais attention si vous utilisez le serveur TomEE, le nom du contexte doit correspondre au nom de l’artéfact déployé (ici ce serait probablement « tennis-1.0-SNAPSHOT ».
